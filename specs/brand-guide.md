## 3. Voice & Copy Rules

### Principles
- Short sentences. One idea per sentence.
- No adverbs. No filler words.
- Concrete before abstract — lead with what you do, then why it matters.
- Kaizen is a philosophy, not a buzzword. Use it deliberately, not decoratively.
- The reader is an executive. Write at that altitude.

### Approved Vocabulary
`compound` · `architecture` · `leverage` (sparingly) · `foresight` · `precision` · `deliberate` · `foundation` · `clarity` · `permanence` · `hybrid intelligence` · `signal` · `compounding` · `durable`

### Banned Vocabulary
`unlock` · `empower` · `revolutionize` · `seamless` · `robust` · `cutting-edge` · `world-class` · `next-generation` · `synergy` · `innovative` · `game-changing` · `disruptive` · `holistic`

## 5. Visual Identity System

### Color Palette

| Token | Value | Usage |
|-------|-------|-------|
| `--bg` | `#0D1117` | Primary background — deep navy, not pure black |
| `--bg2` | `#101418` | Alternate section background |
| `--surface` | `#161B24` | Card / form field background |
| `--text` | `#F0EDE6` | Primary text — warm off-white, not pure white |
| `--text-dim` | `rgba(240,237,230,0.50)` | Secondary text, body copy |
| `--text-mute` | `rgba(240,237,230,0.28)` | Tertiary / labels / captions |
| `--gold` | `#B8985A` | THE accent — see Gold Discipline below |
| `--gold-dim` | `rgba(184,152,90,0.35)` | Borders, divider rules, active nav border |
| `--gold-mute` | `rgba(184,152,90,0.12)` | Subtle dividers, grid lines, hairlines |
| `--nav-bg` | `rgba(13,17,23,0.92)` | Nav frosted background |
| `--nav-bg-strong` | `rgba(13,17,23,0.98)` | Nav background on scroll |

**Never use hardcoded hex values. Always reference CSS tokens.**

### Gold Discipline
`--gold` (#B8985A) appears in exactly these contexts — and nowhere else:

1. The wordmark `IQ` suffix
2. Overline rules and overline label text
3. Kana annotations (改善, 前進)
4. Primary CTA button fill
5. Logo / brand mark strokes

If you are tempted to use gold in a sixth context — don't. Remove one first.

### Typography

**Display font:** Cormorant Garamond (Google Fonts)
`ital,wght@0,300;0,400;0,600;1,300;1,400`

**Body font:** Inter (Google Fonts)
`wght@300;400;500`

**Google Fonts import:**
```html
<link rel="preconnect" href="https://fonts.googleapis.com"/>
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin/>
<link href="https://fonts.googleapis.com/css2?family=Cormorant+Garamond:ital,wght@0,300;0,400;0,600;1,300;1,400&family=Inter:wght@300;400;500&display=swap" rel="stylesheet"/>
```

**Never use:** Arial, Roboto, system-ui as visible text fonts.
**Never use:** font-weight 700 or 800. This design uses contrast, not weight, for hierarchy. Max weight is 600.

### Type Scale

| Level | Size | Font | Weight | Notes |
|-------|------|------|--------|-------|
| Hero H1 | `clamp(58px, 6.5vw, 84px)` | Cormorant | 600 | line-height 0.92 |
| Page H1 | `clamp(48px, 6vw, 80px)` | Cormorant | 600 | |

**Italic rule:** Italic is always Cormorant Garamond, always weight 300. Use it for the final word or phrase of a headline — never mid-sentence.

### Spacing System

| Context | Value |
|---------|-------|
| Section padding desktop | `120px` top/bottom, `clamp(24px, 4vw, 48px)` sides |
| Section padding mobile | `72px` top/bottom, `24px` sides |

### Border & Line Language

| Rule | Value |
|------|-------|
| All structural borders | `0.5px` — never 1px |
| Border color (default) | `var(--gold-mute)` |
| Border color (active/hover) | `var(--gold-dim)` |
| Border radius | `2px` maximum — this design does not round things |
| Divider rules | `0.5px`, `28–48px` wide, `var(--gold-dim)` |

### Motion System

```css
--ease: cubic-bezier(0.22, 1, 0.36, 1)   /* fast out, no bounce */

Hover transitions:    0.2s
Section transitions:  0.4s
Scroll reveal:        0.7s — opacity 0→1 + translateY(24px→0)
SVG pulse:            2.6s — scale 0.5→2.6, opacity 0.75→0
```

**Never use:** bounce, spring, or overshoot animations. The motion is precise and swift.

## 6. Layout System

### The Two-Column Grid
Every major section is two columns:
- **Left:** argument — text, narrative, headline
- **Right:** proof — symbol, visual, diagram, form
- Separated by a `0.5px var(--gold-mute)` hairline border (not a gap)
- Columns collapse to single stack on mobile (≤1100px)

### Hierarchy Rule
Every section has exactly **one** loudest element. Not two. One.

### Overline Pattern (Required)
Every section opens with an overline:
```html
<div class="overline">
  <span class="overline-rule"></span>   <!-- 28px gold hairline -->
  <span>Label Text</span>              <!-- 10px uppercase gold -->
</div>
```
No section heading appears without an overline above it.

### Responsive Breakpoints

| Breakpoint | Changes |
|-----------|---------|
| `≤1100px` | Hero becomes single column; AI cards stack; vertical labels hidden |
| `≤960px` | Nav collapses to hamburger overlay; form/grid layouts simplify |
| `≤720px` | Section padding drops to 72px/24px; buttons go full-width; font sizes adjust |

## 8. Landing Page Template (Agent-Ready)

Use this section order for any campaign landing page. The design system (tokens, fonts, nav, footer, CTA) is identical to the main site. Only the hero copy, body copy, and featured capabilities vary per campaign.

### Section Order
```
1. Nav           — identical component, links back to main site
2. Hero          — campaign H1, kana, subhead, body, primary CTA
3. Philosophy    — one section establishing the problem/tension (adapt Narrative section)
4. Capabilities  — 2–3 of the four practice areas most relevant to campaign audience
5. How We Do It  — three-step method (can reuse exactly)
6. CTA           — identical component
7. Footer        — identical component
```

### What Stays Identical
- All CSS tokens (copy `style.css` or link to it)
- Font imports (Cormorant Garamond + Inter, same weights)
- Nav component (exact markup)
- CTA section (exact markup and copy)
- Footer component (exact markup)
- Overline pattern on every section
- All border/radius/spacing rules

### What Can Vary Per Campaign
- Hero H1 and body copy (must follow voice rules)
- Kana stays but context copy adapts
- Which 2–3 capabilities to feature (omit irrelevant ones)
- Page `<title>`, meta description, canonical URL
- Schema `@type` stays `ProfessionalService`; update `url` to landing page URL
