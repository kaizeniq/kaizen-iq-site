# Design Vision — Kaizen IQ

This file expands the design direction from `CLAUDE.md`. It is not a replacement for `CLAUDE.md`; when there is any conflict, `CLAUDE.md` wins.

---

## North Star

Kaizen IQ should feel deliberate, restrained, and precise. The design language combines Palantir-grade intelligence, Apple-level restraint, wealth signaling, and Japanese craftsmanship.

The test for every section:
1. Does the layout still feel intentional if the copy is removed?
2. Does any element look default?
3. Does the composition feel generated?
4. Is there one clear visual hierarchy?
5. Is gold used only where it earns attention?

If a section looks generic, rebuild it.

---

## Visual Personality

- Bold but minimal.
- Asymmetric but balanced.
- Quiet, technical, and permanent.
- Premium without decoration.
- Crafted, not templated.

Avoid:
- Stock photography.
- Blue gradients or purple accents.
- Rounded cards with drop shadows.
- Centered hero compositions.
- Decorative UI patterns that do not carry information.
- Any visual decision that feels like a SaaS template.

---

## Locked Design Decisions

- Color palette is final. Use only tokens from `CLAUDE.md`.
- Font pair is final: Cormorant Garamond and Inter.
- Border radius is 2px maximum.
- Structural borders are 0.5px.
- Gold is an accent, not a theme.
- Hero layout is approved: two-column, asymmetric, headline left, logo and principles right.
- Visuals must be bespoke SVG diagrams, not stock images.

---

## Layout Language

Use a two-column grid as the primary structure. The left column usually carries the argument. The right column carries symbol, proof, diagram, or supporting structure.

Use hairline dividers rather than large empty gaps when sections need separation. The grid should feel engineered, not decorative.

Every section needs one dominant element:
- Hero: headline.
- How We Do It: ghost step number.
- Solutions: solution name.
- CTA: main heading.

Do not let two elements compete for the same level of attention.

## Page Intent

### Home

The home page should introduce Kaizen IQ as a strategic data, AI, and growth partner. It should move from positioning to method to proof of capability to conversion.

Required flow:
1. Nav.
2. Hero.
3. Narrative.
4. How We Do It.
5. AI in Action.
6. Strategic Marketing.
7. CTA.
8. Footer.

### Solutions

The solutions page should show concrete capability areas through alternating text and SVG diagram blocks.

Required solution blocks:
1. Data Systems & Architecture.
2. Data Science, ML & AI.
3. Strategic Insight & Growth.
4. Market Impact & Positioning.

### Contact

The contact page should feel like starting a serious conversation, not filling out a generic form. Keep it simple, intentional, and direct.

Required fields:
- Name.
- Company.
- What you're working on.
- How you found us.

CTA label: "Send the signal".

---

## Gold Discipline

Gold should appear only in the approved contexts from `CLAUDE.md`:
1. Wordmark "IQ" suffix.
2. Overline rules and labels.
3. Kana annotations.
4. Primary CTA fill.
5. Logo or enso SVG strokes.
6. Tag borders on solution cards.

Before adding gold, ask whether one of those contexts already covers the need. If not, use text, opacity, spacing, or structure instead.
